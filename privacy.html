Who We Are

Block Frenzy is developed and published by Cloudzzy Ltd. When this policy says "we", "us", or "our", it refers to Cloudzzy Ltd as the data controller for the Block Frenzy mobile application on iOS and Android.

How We Use Your Data

We process your data for these specific purposes only:

Gameplay & Progression

Scores, stats, and game state are used to save your progress, display your profile, track career statistics, and resume interrupted sessions.

Leaderboards & Cloud Sync

Your display name and scores are synced to Cloud Firestore so you can appear on leaderboards and restore progress across devices. Cloud sync uses smart conflict resolution — your highest stats always win.

AI-Powered Features

When AI features are enabled, non-personal game data (board layout, piece positions, statistics) is sent to our backend to generate hints, commentary, and post-game analysis. See the AI section for full details.

Advertising

We serve interstitial and rewarded ads via Google AdMob. Ads are never shown during active gameplay. AdMob may use device identifiers for ad personalisation, which you can control in your device settings.

Purchases

In-app purchases (Remove Ads, Pro subscription) are processed entirely by Apple, Google, and RevenueCat. We receive entitlement status only — never your payment details.

Analytics

Anonymous gameplay analytics (session length, games played, feature usage) help us understand how players interact with the game and improve the experience.

AI & Large Language Models

Transparency

Block Frenzy uses AI to power gameplay features. We believe you should know exactly what data is sent, where it goes, and how to control it.

AI Gameplay Features (Hints, Commentary, Analysis)

When enabled, the app sends non-personal game data — your board layout, available pieces, score, and move history — to our Firebase-authenticated backend. This data is used to generate real-time hints, live commentary, smart piece suggestions, and post-game analysis. No personal information (name, email, location) is included in these requests.

AI Avatar Generation (Pro Only, Opt-In)

Pro subscribers can generate a custom avatar from a photo. When you choose to use this feature:

  • Your selected photo is sent as base64 data to a Firebase Cloud Function
  • The Cloud Function forwards it to OpenAI's gpt-image-1 API for stylised avatar generation
  • The original photo is not stored after processing — only the generated avatar image is kept in Firebase Cloud Storage
  • Avatar generation is rate-limited to 3 per day per user
You're in control

All AI features can be fully disabled in Settings → AI Assistant. When disabled, zero data is sent to any AI service. Avatar generation is entirely opt-in and requires explicit action.

Data Storage & Security

Local Data

All on-device data (game state, preferences, profile) is stored using Hive with AES-256 encryption. Sensitive values such as auth tokens are stored in platform-native secure storage (Keychain on iOS, EncryptedSharedPreferences on Android). No plaintext sensitive data is written to disk.

Cloud Data

Cloud data is stored in Google Cloud Firestore within the Firebase ecosystem, protected by Google Cloud's infrastructure security, encryption at rest (AES-256), and encryption in transit (TLS 1.3). Access to cloud data requires Firebase Authentication.

App Integrity

The app performs basic device integrity checks to protect the fairness of leaderboards. No root/jailbreak data is transmitted — only a pass/fail signal is logged.

Data Sharing

Our Promise

We do not sell, rent, or trade your personal data. Ever. Your data is never shared with data brokers, marketing companies, or any third party for their own commercial purposes.

Data is shared only with the third-party service providers listed above, solely for the purpose of delivering the app's functionality (analytics, ads, purchases, leaderboards, AI features). Each provider receives only the minimum data required to perform their function.

Children's Privacy

Block Frenzy includes a Kids Mode designed for younger players. Kids Mode is protected by a parental gate (a simple maths question) to prevent unsupervised access by very young children.

In Kids Mode:

  • No additional personal information is collected beyond standard gameplay analytics
  • No social features, leaderboards, or user-generated content are available
  • Ads follow the same rules as the rest of the app (never during gameplay)
  • AI features are available only if enabled by a parent in Settings

We do not knowingly collect personal information from children under 13. If you believe a child has provided personal information through the app, please contact us so we can take appropriate action. We comply with COPPA (Children's Online Privacy Protection Act) and applicable international children's privacy laws.

Data Retention

Local data is retained on your device until you uninstall the app or clear app storage.

Cloud data (profile, leaderboard entries, synced game state) is retained until you request account deletion. Upon deletion, all associated Firestore documents and Cloud Storage files are permanently removed within 30 days.

Analytics data is retained according to Firebase Analytics' standard retention policy (14 months for user-level data, aggregated data retained indefinitely).

AI processing data (board states sent for hints/analysis) is transient and not stored after the response is generated.

Avatar photos (sent for generation) are not retained after processing. Only the generated avatar image is stored in Cloud Storage, linked to your account.

App Permissions

Block Frenzy requests only the permissions it needs:

  • Camera / Photo Library — Used only for AI avatar generation (Pro, opt-in). Never accessed without your explicit action.
  • Vibration — Haptic feedback during gameplay. Can be disabled in Settings.
  • Internet — Required for cloud sync, leaderboards, ads, analytics, AI features, and purchases.

Block Frenzy does not request access to your location, contacts, microphone, calendar, or any other sensitive permissions.
